АРГУМЕНТЫ ФУНКЦИИ main()

process_data (тип: bool, по умолчанию: True)
Отвечает за обработку сырых данных из Excel файла. Если True, система проверяет наличие файла real_orders.xlsx и при необходимости конвертирует его в обработанный CSV формат. Если файл CSV уже существует и новее Excel-файла, обработка пропускается автоматически. Установите в False, если хотите пропустить этот этап (например, данные уже обработаны).

train_model (тип: bool, по умолчанию: True)
Запускает обучение модели машинного обучения на подготовленных данных. Если True, модель обучается и сохраняется в файл model_orders.pkl. Если False, обучение пропускается (используется ранее сохранённая модель).

make_forecast (тип: bool, по умолчанию: True)
Запускает генерацию прогноза на будущий период. Если True, модель делает предсказание количества заказов и гостей по часам. Если False, прогноз не генерируется.

evaluate (тип: bool, по умолчанию: True)
Запускает оценку качества модели на тестовых данных. Если True, после обучения модель тестируется и выводятся метрики точности (MAE, R², точность в процентах и другие). Работает только если train_model=True.

from_date (тип: str или datetime, по умолчанию: None)
Дата начала прогноза в формате ГГГГ-ММ-ДД (например, 2026-01-01). Если указана, прогноз начинается с этой даты. Если None, прогноз начинается со следующего часа после последней записи в исторических данных.

to_date (тип: str или datetime, по умолчанию: None)
Дата конца прогноза в формате ГГГГ-ММ-ДД (например, 2027-01-01). Если указана, прогноз заканчивается этой датой. Если None, используется параметр hours_ahead для определения длительности прогноза.

hours_ahead (тип: int, по умолчанию: 168)
Количество часов вперёд для прогноза. По умолчанию 168 часов (7 дней). Используется только если from_date и to_date не указаны. Например, 336 часов = 14 дней, 720 часов = 30 дней.

verbose (тип: bool, по умолчанию: True)
Включает подробный вывод информации в консоль. Если True, система показывает детальные отчёты на каждом этапе (статистика данных, метрики модели, прогноз по часам). Если False, вывод минимален (только ошибки и критические сообщения).

КУДА СОХРАНЯЮТСЯ РЕЗУЛЬТАТЫ

Обработанные данные
Путь: data/processed/processed_orders.csv
Что содержится: Агрегированные данные по часам (datetime, orders_count, guests_count, checks_count, avg_guests_per_check) и погода. Этот файл создаётся из сырого Excel и используется для обучения модели.

Обученная модель
Путь: models/model_orders.pkl
Что содержится: Обученная модель RandomForest, список признаков, метрики качества, коэффициент конверсии гостей (гостей на 1 заказ). Файл используется для генерации прогнозов.

Прогноз
Путь: data/predicted/forecast.csv
Что содержится: Почасовой прогноз на указанный период. Колонки: forecast_datetime, date, hour, day_of_week, is_peak_hour, is_weekend, is_holiday, orders_predicted, orders_with_buffer, guests_predicted, guests_with_buffer. Файл обновляется при каждом запуске прогноза.

СТРУКТУРА ПРОГНОЗА В CSV

forecast_datetime — дата и время прогноза (например, 2026-04-01 12:00:00)
date — дата без времени (например, 2026-04-01)
hour — час суток (0-23)
day_of_week — день недели (0=понедельник, 6=воскресенье)
is_peak_hour — флаг пикового часа (1=обед 12-14 или ужин 18-21, 0=обычный час)
is_weekend — флаг выходного дня (1=суббота или воскресенье, 0=будний)
is_holiday — флаг праздника (1=праздник, 0=обычный день)
orders_predicted — прогнозируемое количество заказов
orders_with_buffer — прогнозируемое количество заказов с буфером +25%
guests_predicted — прогнозируемое количество гостей
guests_with_buffer — прогнозируемое количество гостей с буфером +25%
