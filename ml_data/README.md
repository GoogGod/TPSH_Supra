## Быстрый старт

1. Поместите сырой датасет в data/raw/real_orders.xlsx
Формат: Время открытия | Номер чека | Заказов | Количество гостей

2. Запустите полный пайплайн
python main.py

3. Получите результаты:
   - data/predicted/forecast.csv — прогноз
   - data/predicted/waiter_schedule.csv — расписание

## Аргументы функции main()

process_data (bool, по умолчанию: True)
Отвечает за обработку сырых данных из Excel файла. Если True, система проверяет наличие файла real_orders.xlsx и при необходимости конвертирует его в обработанный CSV формат. Если файл CSV уже существует и новее Excel-файла, обработка пропускается автоматически. Установите в False, если хотите пропустить этот этап (например, данные уже обработаны).

train_model (bool, по умолчанию: True)
Запускает обучение модели машинного обучения на подготовленных данных. Если True, модель обучается и сохраняется в файл model_orders.pkl. Если False, используется ранее сохранённая модель.

make_forecast (bool, по умолчанию: True)
Запускает генерацию прогноза на будущий период. Если True, модель делает предсказание количества заказов и гостей по часам.

evaluate (bool, по умолчанию: True)
Запускает оценку качества модели на тестовых данных. Выводятся метрики точности (MAE, R2, точность в процентах). Работает только если train_model=True.

make_schedule (bool, по умолчанию: True)
Запускает планирование смен официантов на основе прогноза. Если True, создаётся файл waiter_schedule.csv с расписанием для каждого официанта.

incremental_training (bool, по умолчанию: False)
Режим дообучения существующей модели. Если True и файл модели существует, система дообучает модель на новых данных вместо полного переобучения. Используется вместе с папкой data/raw_new для новых данных.

model_type (str, по умолчанию: xgboost)
Тип модели для обучения. Доступные значения: xgboost, random_forest, gradient_boosting.

min_hours_per_waiter (int, по умолчанию: 200)
Минимальная норма часов работы на одного официанта в месяц. Система автоматически масштабирует норму по количеству месяцев в прогнозе (например, 200ч/мес × 3мес = 600ч).

novice_ratio (float, по умолчанию: 0.5)
Доля новичков среди официантов (от 0.0 до 1.0). Например, 0.5 означает 50% новичков и 50% специалистов. Система автоматически распределяет типы официантов.

from_date (str или datetime, по умолчанию: None)
Дата начала прогноза в формате ГГГГ-ММ-ДД. Если указана, прогноз начинается с этой даты. Если None, прогноз начинается со следующего часа после последней записи в исторических данных.

to_date (str или datetime, по умолчанию: None)
Дата конца прогноза в формате ГГГГ-ММ-ДД. Если указана, прогноз заканчивается этой датой. Если None, используется параметр hours_ahead.

hours_ahead (int, по умолчанию: 168)
Количество часов вперёд для прогноза. По умолчанию 168 часов (7 дней). Используется только если from_date и to_date не указаны.

verbose (bool, по умолчанию: True)
Включает подробный вывод информации в консоль. Если True, система показывает детальные отчёты на каждом этапе. Если False, вывод минимален.

force_fresh_weather (bool, по умолчанию: True)
Всегда загружать свежую погоду из API для прогноза. Если False, может использоваться кэшированная погода.

## Структура данных

### Входные данные

Путь: data/raw/real_orders.xlsx

Колонки:
- Время открытия (datetime) — Дата и время чека
- Номер чека (int) — Уникальный номер заказа
- Заказов (int) — Количество позиций в чеке
- Количество гостей (int) — Количество гостей в чеке

### Обработанные данные

Путь: data/processed/processed_orders.csv

Колонки:
- datetime — Дата и время (агрегация по часам)
- orders_count — Количество заказов в час
- guests_count — Количество гостей в час
- checks_count — Количество чеков в час
- avg_guests_per_check — Среднее гостей на чек
- temperature_mean — Средняя температура
- precipitation — Осадки
- is_rainy — Флаг дождя
- is_extreme_weather — Флаг экстремальной погоды
- weather_source — Источник погодных данных

## Выходные файлы

### Обработанные данные

Путь: data/processed/processed_orders.csv

Содержание: Агрегированные данные по часам (datetime, orders_count, guests_count, checks_count, avg_guests_per_check) и погодные данные (temperature_mean, precipitation, is_rainy, is_extreme_weather, weather_source и другие).

### Обученная модель

Путь: models/model_orders.pkl

Содержание: Обученная модель (XGBoost), список признаков, метрики качества, коэффициент конверсии гостей, дата обучения.

### Прогноз

Путь: data/predicted/forecast.csv

Содержание: Почасовой прогноз на указанный период с колонками: datetime, hour, day_of_week, is_peak_hour, is_weekend, is_holiday, orders_predicted, orders_with_buffer, guests_predicted, guests_with_buffer. Прогноз включает сезонную корректировку для месяцев без истории.

### Расписание официантов

Путь: data/predicted/waiter_schedule.csv

Содержание: Расписание для каждого официанта с колонками: date, waiter_id, waiter_num, waiter_type, waiter_type_code, waiter_capacity, shift_type_code, shift_type, work_start, work_end, work_hours, guests_predicted.

## Структура прогноза в CSV

datetime — дата и время прогноза (например, 2026-04-01 12:00:00)

hour — час суток (0-23)

day_of_week — день недели (0=понедельник, 6=воскресенье)

is_peak_hour — флаг пикового часа (1=обед 12-14 или ужин 18-21, 0=обычный час)

is_weekend — флаг выходного дня (1=суббота или воскресенье, 0=будний)

is_holiday — флаг праздника (1=праздник, 0=обычный день)

orders_predicted — прогнозируемое количество заказов (с учётом сезонности)

orders_with_buffer — прогнозируемое количество заказов с буфером +50% и сезонной корректировкой

guests_predicted — прогнозируемое количество гостей (с учётом сезонности)

guests_with_buffer — прогнозируемое количество гостей с буфером +50% и сезонной корректировкой

## Структура расписания официантов

date — дата смены

waiter_id — идентификатор официанта (например, Официант 1)

waiter_num — номер официанта (1, 2, 3...)

waiter_type — тип официанта (Специалист или Новичок)

waiter_type_code — код типа официанта (1=Специалист, 2=Новичок)

waiter_capacity — вместимость официанта (10 или 5 гостей/час)

shift_type_code — код типа смены (1=Полная, 2=Утренняя, 3=Вечерняя, 0=Выходной)

shift_type — тип смены (Полная, Утренняя, Вечерняя, Выходной)

work_start — начало смены (10, 16 или None)

work_end — конец смены (22, 25 или None, где 25=01:00 следующего дня)

work_hours — продолжительность смены в часах (12, 6, 9 или 0)

guests_predicted — прогнозируемое количество гостей в этот день

## Как обучить модель и получить расписание

1. Установить все нужные библиотеки: pip install -r requirements.txt

2. Сырой датасет поместить по пути: data/raw под названием real_orders.xlsx

3. Формат Excel файла должен содержать колонки: Время открытия, Номер чека, Заказов, Количество гостей

4. Запустить файл main.py: python main.py

5. Получить результаты:
   - data/predicted/forecast.csv — прогноз заказов и гостей
   - data/predicted/waiter_schedule.csv — расписание смен официантов
