# Стек и структура

---

## Контекст проекта

**Supra** — корпоративная автоматизированная система планирования трудовых ресурсов.

### Цель разработки
Обеспечить автоматизированное управление графиками работы персонала на основе прогнозных данных о загруженности объектов, минимизировать участие менеджеров в рутинных операциях по составлению расписаний.

### Первичный scope
На текущем этапе система разрабатывается **только для официантов** с возможностью дальнейшего расширения на другие категории персонала.

### Основные пользовательские роли
- **Сотрудник**
- **Менеджер**
- **Администратор**

### Подход к разработке
Проект ведется по принципу **Docs First, Code After**:
1. Сначала формируется документация по стеку, структуре, модулям, activity/use-case сценариям.
2. Затем реализуется код в соответствии с утвержденной документацией.
3. Все изменения сначала попадают в ветку **develop**, затем после проверки — в **main**.

### Организация разработки
- Репозиторий: **приватный Git**
- Ветки:
  - `main` — стабильная версия
  - `develop` — основная ветка разработки
- Доска проектирования: **Miro**
- Архитектура репозитория:
  - `front/`
  - `backend/`
  - в будущем возможно добавление дополнительных директорий (`docs/`, `infra/`, `scripts/` и т.д.)

### Назначение аналитического блока
Прогнозно-аналитический модуль должен:
- собирать исторические данные по загрузке объекта;
- учитывать:
  - день недели,
  - сезонность,
  - праздничные даты,
  - исторические пики/спады;
- формировать прогноз требуемой численности персонала на каждую смену;
- стремиться к точности прогноза **не ниже 85%**.

### Бесплатные/условно-бесплатные платформы для сервера
Для первоначального этапа рекомендуется рассмотреть:
- **Render** — удобен для Django + PostgreSQL, простая настройка через Git
- **Railway** — удобен для быстрого старта, но нужно следить за лимитами
- **PythonAnywhere** — подходит для простых Django-проектов, но менее гибок
- **VPS с бесплатным периодом / студенческими программами** — если нужен полный контроль

### Рекомендация по старту
Для MVP лучше использовать:
- **Backend**: Render / Railway
- **Database**: PostgreSQL
- **Frontend**: Vercel / Netlify / тот же Render
- **Домен**: отдельный домен с HTTPS
