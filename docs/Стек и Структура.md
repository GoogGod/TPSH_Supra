# Стек и Структура (актуально)

## Технологический стек

### Backend
- Python 3.12+
- Django
- Django REST Framework
- SimpleJWT (auth)
- drf-spectacular (OpenAPI/Swagger)
- PostgreSQL (прод), SQLite (локально)

### ML слой
- Отдельный модуль `ml_data/`
- pandas / numpy / sklearn (внутри пайплайна)
- Прогноз + scheduler/planner для смен
- Хранение артефактов по запускам и venue

### Frontend
- Vue 3 + Vite (SPA)
- Общение с backend через `/api/v1/...`

### Deploy
- Docker
- Render (`render.yaml`)

## Структура проекта

```text
Supra/
├─ backend/
│  ├─ core/                     # settings, root urls, SPA fallback
│  ├─ users/                    # auth, пользователи, роли
│  ├─ shifts/                   # venues, schedules, slots, draft/publish
│  ├─ forecasting/              # ML runs, прогнозы, загрузка данных
│  ├─ user_notifications/       # уведомления и подтверждения
│  └─ ...
├─ ml_data/
│  ├─ main.py                   # основной ML pipeline
│  ├─ scheduler.py              # планировщик смен
│  ├─ data/
│  │  ├─ raw/
│  │  ├─ processed/
│  │  └─ predicted/
│  │     └─ runs/
│  │        └─ venue_<id>/run_<id>/
│  │           ├─ forecast.csv
│  │           └─ waiter_schedule.csv
│  └─ models/
│     └─ model_orders.pkl
├─ front/                       # SPA клиент
├─ docs/
└─ Dockerfile / render.yaml
```

## Ключевые изменения в текущей версии

- Генерация расписания идет через `POST /api/v1/schedule/generate/`.
- Артефакты прогноза и расписания изолированы по `venue` и `run`.
- Добавлено массовое редактирование черновика entries.
- В API расписания добавлены признаки `employee_pro` / `employee_noob` из сгенерированных слотов.
- Добавлены метрики нехватки сотрудников в ответ генерации.
